# McuPilot Setup — 功能规格

> 给前端开发 / AI 生成用的功能需求书。只描述内容和流程，不含任何视觉设计。

---

## 窗口信息

- 标题：McuPilot — 项目配置向导
- 尺寸：固定宽高（建议 680×620），不可缩放
- 包含两个页面，通过"上一步 / 下一步"切换

---

## 页面一：工程设置

### 区块 1.1 — 工程路径

**内容：**
- 标签文字："工程路径"
- 一个文本输入框（占位提示："输入或浏览 Keil 工程目录"）
- 一个"浏览"按钮（点击弹出系统文件夹选择对话框）

**交互：**
- 用户手动输入路径或点击浏览选择文件夹
- 路径变更后，自动触发区块 1.2 的环境检测

### 区块 1.2 — 环境

**内容：**
- 标签文字："环境"
- 三个水平并列的状态项：

| 检测项 | 未检测时显示 | 检测通过时显示 |
|--------|-------------|---------------|
| 芯片型号 | 灰色占位 `--` | 绿色状态 + 具体型号，如 `HC32F460` |
| Keil MDK | 灰色占位 `--` | 绿色状态 + 安装路径，如 `C:\Keil_v5\UV4` |
| J-Link | 灰色占位 `--` | 绿色状态 + 序列号和电压，如 `SN 801025438, 3.31V` |

**交互：**
- 区块 1.1 的路径变化时自动刷新
- 检测失败时显示红色状态 + 错误原因（如"未检测到 J-Link"）
- 全部通过后才能点击"下一步"

### 区块 1.3 — 可热修参数

**内容：**
- 标签文字："可热修参数"
- 一个表格，列：变量名、类型、默认值、删除按钮
- 删除按钮为 × 图标
- 表格下方一个"+ 添加参数"按钮

**类型下拉选项：**
`uint8_t`、`uint16_t`、`uint32_t`、`int8_t`、`int16_t`、`int32_t`、`float`

**交互：**
- 点"+ 添加参数"在表格末尾新增一行（变量名默认 `new_var`，类型默认 `uint32_t`，默认值默认 `0`）
- 点 × 删除该行
- 允许 0 行（没有热修参数的情况）
- 添加和删除有过渡效果

### 区块 1.4 — AI 客户端

**内容：**
- 标签文字："AI 客户端"
- 三个复选框：`Claude Code`、`Roo Code`、`Codex CLI`

**交互：**
- 至少勾选一个

## 页面二：确认部署

### 区块 2.1 — 部署摘要

**内容：**
- 标签文字："部署摘要"
- 四行信息：

| 标签 | 值来源 |
|------|-------|
| 工程 | 页面一的工程路径 |
| 芯片 | 页面一检测到的芯片型号 |
| 参数 | 页面一参数表的行数 + 变量名列表 |
| 客户端 | 页面一勾选的 AI 客户端名称 |

### 区块 2.2 — 部署按钮

**内容：**
- 一个"开始部署"按钮，居中

**交互：**
- 点击后按钮变灰禁用，显示"部署中..."
- 开始按顺序执行六个步骤
- 六步全部完成后按钮变为"完成"

### 区块 2.3 — 进度

**内容：**
- 标签文字："进度"
- 一个横向进度条（对应 6 步，每步约 16.7%）
- 六个步骤的状态列表：

| 编号 | 步骤名称 | 状态 |
|------|---------|------|
| ① | 生成 hil_config_user.h | 等待 → 进行中 → 完成/失败 |
| ② | 复制 HIL/RTT 到工程 | 同上 |
| ③ | COM 注入 Keil 工程 | 同上 |
| ④ | 编译工程 | 同上，完成后显示 (N errors, N warnings) |
| ⑤ | 解析内存字典 | 同上，完成后显示 (N 符号) |
| ⑥ | 注册 MCP | 同上 |

**交互：**
- 六步顺序执行，不可并行
- 步骤进行中：显示加载指示器
- 步骤完成：绿色勾 + 附加信息
- 步骤失败：红色叉 + 错误详情可展开，提供"重试此步"按钮
- 某步失败后，依赖它的后续步骤自动跳过（标记为"跳过"），不依赖的继续执行
- 进度条平滑过渡

---

## 部署完成弹窗

**触发时机：** 六个步骤全部执行完毕（成功或部分失败）

**内容：**
- 标题："部署完成"
- 如果 COM 步骤失败，提示用户手动将 HIL/ 和 RTT/ 的 .c 文件拖入 Keil 工程树
- main.c 集成代码示例（C 语言代码块）
- 脚注："完整范例见 HIL/example_main.c"
- 一个"关闭"按钮

**交互：**
- 点击遮罩区域或关闭按钮关闭弹窗
- 弹窗出现时有入场动画

---

## 依赖链

```
① 生成 hil_config_user.h         ← 独立
② 复制 HIL/RTT 到工程             ← 独立
③ COM 注入 Keil 工程              ← 依赖②
④ 编译工程                        ← 依赖③（文件需在工程编译列表中）
⑤ 解析内存字典                    ← 依赖④（需要 .map / .axf）
⑥ 注册 MCP                        ← 独立
```

## 底部导航

- 页面一：`[禁用] ← 上一步` 和 `下一步 →`
- 页面二：`← 上一步` 和 `关闭`
- 部署进行中时所有按钮禁用
- 顶部有步骤指示器（1 工程设置 / 2 部署），当前步骤高亮
